# Known Limitations

Klemm is alpha software.

## Current Limits

- Klemm does not automatically control unmanaged agents.
- Klemm does not directly control Chrome, email, calendars, or ordinary apps.
- Browser/email/docs are context sources only through supported import/observation paths.
- Raw imported context is not authority until reviewed or pinned.
- Codex is the strongest adapter path.
- Claude Code support depends on installed hooks and live Claude usage.
- Shell supervision is strong for commands launched through Klemm.
- Browser/MCP agent proof depends on compatible envelope reporting.
- Endpoint Security blocking is capability-gated and fails safely when unavailable.
- Hosted sync is encrypted bundle transport, not a hosted plaintext brain.
- External security review is still needed before broad alpha distribution.
- Dogfood exports and saved-me reports are local evidence aids, not proof that every adapter will behave correctly in every future session.

## Honest Product Boundary

Klemm is a local authority layer for AI agents. It watches agents, keeps them on mission, queues risky actions, and explains decisions. It is not a fully autonomous operating system, not an AI lawyer, and not a universal app-control layer.
